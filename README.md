# 🛡️ SmartShield AI

**Hybrid Explainable Spam & Phishing Detection System using NLP, LSTM, and Ensemble Learning**

SmartShield AI detects spam and phishing messages across SMS, Email, WhatsApp, Telegram, and
social media by combining a classical NLP pipeline (TF-IDF + Logistic Regression) with a deep
learning pipeline (Embedding + LSTM) in a weighted ensemble — and then **explains** every
verdict using keyword intelligence, URL risk analysis, emotion/manipulation detection, and
named entity recognition.

## Features

- **Hybrid AI Engine** — TF-IDF + Logistic Regression, and Embedding + LSTM, fused via a
  confidence-weighted ensemble. When the two models disagree significantly, the message is
  flagged **Suspicious** instead of a risky hard Spam/Ham call.
- **Explainable AI** — every prediction ships with human-readable reasons: which keyword
  categories fired (prize, urgency, threat, finance, OTP, government, courier, clickbait...),
  which emotions/manipulation tactics were used, which URLs are risky and why, and which
  entities (money, phone, email, bank names) were found.
- **URL Risk Engine** — flags IP-based URLs, known shorteners (bit.ly, tinyurl...), suspicious
  TLDs, excessive sub-domains, and non-HTTPS links.
- **Keyword Intelligence** — 10 curated categories covering prize/lottery, urgency, account
  threats, banking, OTP/credentials, government/tax, courier scams, clickbait, money values,
  and romance-scam language.
- **Named Entity Recognition** — uses spaCy when available, with an automatic regex-based
  fallback (money, phone, email, bank names) so the app always runs.
- **Analytics Dashboard** — class distribution, message-length histograms, word clouds, top
  spam/ham words, confusion matrices, ROC curves, and model comparison charts (Plotly).
- **Modern Dark UI** — a 6-page Streamlit app (Home, Spam Detection, Dashboard, Model
  Comparison, Dataset Analysis, About) with a custom cybersecurity-themed dark design.

## Architecture

```
Message Input (SMS / Email / WhatsApp / Telegram / Social)
        │
        ├──▶ NLP Preprocessing (clean → tokenize → lemmatize)
        │
        ├──▶ Model 1: TF-IDF + Logistic Regression ──▶ P(spam)
        ├──▶ Model 2: Embedding + LSTM + Sigmoid   ──▶ P(spam)
        │
        ├──▶ Ensemble Fusion (weighted avg + disagreement check)
        │
        ├──▶ Explainability Layer
        │       (keyword intelligence · URL risk · emotion analysis · NER)
        │
        └──▶ Verdict: Spam / Suspicious / Ham (Safe) + Reasons
```

See `assets/architecture.png` for the visual diagram.

### Ensemble Logic

```python
final_score = (logistic_confidence * 0.4) + (lstm_confidence * 0.6)

if abs(logistic_confidence - lstm_confidence) >= 0.35:
    verdict = "Suspicious"          # models disagree — flag for review
elif final_score >= 0.60:
    verdict = "Spam"
elif final_score >= 0.40:
    verdict = "Suspicious"
else:
    verdict = "Ham (Safe)"
```

Example (matches the assignment spec):
`Logistic Regression = 91%`, `LSTM = 96%` → **Final Ensemble Score = 94% → Spam**

## Installation

**Requirements:** Python 3.10 – 3.12

```bash
# 1. Clone the repository
git clone https://github.com/your-username/SmartShieldAI.git
cd SmartShieldAI

# 2. Create a virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Download NLTK data (done automatically on first run, or manually):
python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab'); nltk.download('stopwords'); nltk.download('wordnet'); nltk.download('omw-1.4')"

# 5. (Optional) enable full spaCy NER instead of the regex fallback
pip install spacy
python -m spacy download en_core_web_sm
```

### Train the Models

The trained model artifacts are not committed to the repo (they're regenerated locally / in CI).
Run the two training scripts once before launching the app:

```bash
python src/train_ml_model.py     # trains Logistic Regression + TF-IDF  (~few seconds)
python src/train_lstm_model.py   # trains the LSTM model                (~1-2 minutes on CPU)
```

This creates `models/logistic_model.pkl`, `models/tfidf.pkl`, `models/tokenizer.pkl`,
`models/lstm_model.keras`, and `models/metrics.json`.

Alternatively, open and run `notebooks/NLP_Model.ipynb` and `notebooks/LSTM_Model.ipynb` for a
step-by-step, annotated walkthrough of the same training process with visualizations.

## Deployment

### Streamlit Community Cloud

1. Push this repository to GitHub (see steps below).
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub.
3. Click **New app**, select your repo, branch `main`, and main file `app.py`.
4. **Important:** Streamlit Cloud does not run your training scripts automatically. Either:
   - Commit the trained `models/*.pkl`, `models/*.keras`, and `models/metrics.json` files to
     the repo (simplest for a class project), **or**
   - Add a `postBuild` / startup step that runs `train_ml_model.py` and `train_lstm_model.py`
     before the app starts (slower cold start, but keeps the repo lightweight).
5. Click **Deploy**. The app will be live at `https://<your-app-name>.streamlit.app`.

### Local Docker (optional)

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
RUN python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab'); nltk.download('stopwords'); nltk.download('wordnet'); nltk.download('omw-1.4')"
RUN python src/train_ml_model.py && python src/train_lstm_model.py
EXPOSE 8501
CMD ["streamlit", "run", "app.py", "--server.port=8501", "--server.address=0.0.0.0"]
```
